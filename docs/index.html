OneTwo API Documentation

Welcome to the OneTwo API. If you have any questions or run into a technical issue, please email me at johnathon@seesykes.codes

Basics

All requests must be made to the following url: https://onetwo-server.herokuapp.com/api/ followed by the endpoint you wish to make the request to. The body of each request must be sent in JSON format unless otherwise stated.

Request Validation

All requests, except for those made to the /login and /signup endpoints must be validated with an authentication token placed in the header of the request. User Signup and Login

Signup

POST /signup

Creates a user based on information sent via the post request.

Here is an example of a request with a JSON body and its expected result: 

                    
                    { 
                        "firstName": "Don", 
                        "lastName": "Ovan", 
                        "userName": "LegenD10", 
                        "email": "CozmoLover@LAGalaxy.com", 
                        "password": "FirstLeonFive57"
                    }
                    
                
                        
                            //response from server
                         { 
                             "message": "user created", 
                             "data": 
                             { 
                                 "firstName": "Don", 
                                 "lastName": "Ovan",  
                                 "userName": "LegenD10", 
                                 "email": "CozmoLover@LAGalaxy.com"
                            } 
                    }

Fields

All fields are required.

firstName: First name of the user.

lastName: Last name of the user.

userName: Desired username for the user.

email: Email address for the user.

password: Password for user.

Additional info

Passwords are not saved as plain-text, but hashed prior to saving. Passwords are not returned by the server.

Login

POST /login

Authenticates the user and passes a JSON web token. You will use the returned token to authenticate additional requests made to the API.

Example request and expected response: 
                    
                         { 
                            "email": "CozmoLover@LAGalaxy.com", 
                            "password": "FirstLeonFive57" 
                        }
                    
                
                    
                         { 
                            "success": true, 
                            "message": "Logged in successfully.", 
                            "token":   "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
                        }
