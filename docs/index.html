OneTwo API Documentation

Welcome to the OneTwo API. If you have any questions or run into a technical issue, please email me at johnathon@seesykes.codes

Basics

All requests must be made to the following url: https://onetwo-server.herokuapp.com/api/ followed by the endpoint you wish to make the request to. All requests, except requests to the /signup or /login endpoints, must be valided or you will get the following response. 

                        
                    { 
                        success: false, 
                        message: "No token provided. Access denied." 
                    }
Each request's body must be sent in JSON format unless otherwise stated.

User Signup and Login

Signup

POST /signup

Creates a user based on information send via the post request.

Here is an example of a request with a JSON body and it's expected result 

                    
                    { 
                        "firstName": "Don", 
                        "lastName": "Ovan", 
                        "userName": "LegenD10", 
                        "email": "CozmoLover@LAGalaxy.com", 
                        "password": "FirstLeonFive57"
                    }
                    
                
                        
                            //response from server
                        { 
                             "message": "user created", 
                             "data": { "__v": 0, 
                             "firstName": "Don", 
                             "lastName": "Ovan", 
                             "userName": "LegenD10", 
                             "email": "CozmoLover@LAGalaxy.com", 
                             "password": "$2a$08$ZJxvMbkwk4lhFipjh3cXW.J3qc4M0.vnJcLUaiMsbevmKdGZJ8NrC", 
                            "_id": "5ade33556e8f060004a9cc27",
                            "_checkIns": [] } 
                        }

Fields

All fields are required.

firstName: First name of the user.

lastName: Last name of the user.

userName: desired username for the user.

email: email address for the user.

password: password for user.
